# Техническая архитектура

## Общий обзор

AI Classifieds Ad Generator — это веб-приложение на Next.js (App Router), использующее гибридную архитектуру. Клиентская часть отвечает за UI и валидацию, а серверная часть (API Routes) управляет взаимодействием с AI и базой данных.

## Ключевые технические решения

### 1. Серверная генерация (API Routes)

**Почему**: Переход с клиентской генерации на серверную позволил скрыть API ключи и улучшить безопасность.

**Как работает**:
- Клиент отправляет POST запрос на `/api/generate`.
- Сервер использует `GOOGLE_API_KEY` для обращения к Gemini.
- Реализована логика повторных попыток (Retry) для обработки ошибок 503 и 429.

### 2. Интеграция с Supabase

Все успешные генерации асинхронно сохраняются в базу данных:
- Таблица: `generated_ads`
- Поля: `id`, `category`, `input_data` (JSON), `output_text`, `tone`, `created_at`.
- Изображения (base64) удаляются из `input_data` перед сохранением для экономии места.

### 3. Отказоустойчивость (Retry Logic)

В API реализован механизм экспоненциальной задержки (Exponential Backoff):
- При ошибках 503 (Overloaded) или 429 (Too Many Requests).
- 3 попытки с интервалами 1s, 2s, 4s.
- Автоматическое переключение моделей (fallback) при необходимости.

### 4. Безопасность

- **API Keys**: Ключи Google и Robokassa хранятся только на сервере.
- **Supabase**: Используется `NEXT_PUBLIC_SUPABASE_ANON_KEY` для клиента и сервисные роли при необходимости (на сервере).
- **Rate Limiting**: Ограничение запросов к API по IP-адресу для предотвращения abuse-а (10 запросов в 10 минут).
- **Referer Validation**: Проверка `Referer` хедера для защиты от CSRF атак.

### 5. Платежная система (Robokassa)

**Почему**: Для монетизации добавлена интеграция с платежной системой Robokassa.

**Как работает**:
- `robokassaService.ts` отвечает за генерацию URL для оплаты и проверку подписей в коллбэках.
- Созданы API-маршруты для обработки коллбэков от Robokassa (`/api/payment/result` и `/api/payment/success`).
- После успешной оплаты, на счет пользователя зачисляются кредиты.

### 6. Система кредитов

**Почему**: Для монетизации и контроля использования ресурсов введена система кредитов.

**Как работает**:
- **База данных**: В Supabase созданы таблицы `user_credits` и `credit_transactions` для отслеживания баланса и истории операций. Каждому новому пользователю начисляется 3 бесплатных кредита.
- **Списание и начисление**: Логика реализована в `creditService.ts` через RPC-вызовы к Supabase. Кредиты списываются при генерации и оптимизации, и начисляются после успешной оплаты.
- **Отображение**: Баланс кредитов управляется через `creditStore.ts` (Zustand) и отображается в интерфейсе.
